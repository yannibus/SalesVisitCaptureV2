# Sales Visit Capture V2

> AI-powered sales visit preparation, capture, and action plan materialization for Salesforce.
> Built for **Solution Engineers**, **presales**, and **field sales reps**.

Turn every sales meeting into a structured CRM artifact тАФ before, during, and after the visit тАФ with Einstein GenAI doing the heavy lifting.

---

## тЬи What's inside

**Three AI chained experiences, one sales motion:**

| Stage | Prompt | Output |
|---|---|---|
| **Before** тАФ Prep the demo | `Prep_Demo_Grounding` | 5 grounding fields (company profile, pain points, hooks, objections, success criteria) |
| **Before** тАФ Prep the visit | `Prepare_Visit_Brief` | A tactical HTML brief scannable in 45 seconds (context, talking points, opening hooks, objections, success criteria) |
| **After** тАФ Capture the visit | `Generate_Visit_Report` | A HTML executive summary + a structured action plan (max 10 tasks & events) |

Plus a **LWC-based cockpit** (KPIs, action queues, analytics) and an **immersive 3-tab Visit Workspace** (Preparation тЖТ Execution тЖТ Outcome) that replaces the standard record page.

## ЁЯОп Architecture

### ЁЯЕ░я╕П Component A тАФ Demo prep (`Demo_Context__c`)

A lightweight object that acts as the **brief sheet for an upcoming demo or meeting**. The SE enters the target account, meeting type, goal, personas, and a few notes. One click on the Quick Action **"Generate Grounding"** triggers `Prep_Demo_Grounding` (Einstein Flex, GPT-4 Omni, JSON output) which writes 5 grounded AI fields:

- **Company Profile** тАФ market/positioning/strategic stakes
- **Pain Points** тАФ likely industry pains
- **Conversation Hooks** тАФ high-impact opening angles
- **Objection Map** тАФ anticipated objections + counter-arguments
- **Success Criteria** тАФ measurable demo success signals

The grounding is reused as **context** by Component B and by the Visit Brief prompt.

### ЁЯЕ▒я╕П Component B тАФ Visit capture (`Visit_Capture__c`)

The central record of a sales visit. It combines:

- **Lookups**: VideoCall (In-Person Meeting / ECI), Demo_Context (prepared grounding), Account, Contact, Opportunity, and a soft-link to the source Event
- **Capture fields**: voice transcript (manual or dictation), free notes, file attachments
- **AI fields**:
  - `AI_Visit_Brief__c` (Rich Text) тАФ pre-meeting tactical brief generated by `Prepare_Visit_Brief`
  - `AI_Report__c` (Rich Text) тАФ HTML post-meeting executive report
  - `AI_Action_Plan__c` (JSON) тАФ structured plan (mix of tasks & events)
  - `Detected_Language__c` тАФ auto-detected language of the content

Three AI actions on the record:

- **Generate visit brief** (tab Preparation) тАФ sharpens the rep before the meeting
- **Generate Report** (tab Execution) тАФ transforms transcript + notes + grounding into a report
- **Materialize Action Plan** (tab Execution) тАФ converts JSON into real Salesforce **Tasks + Events**

### ЁЯдЦ Automation тАФ Auto-create on Event

A record-triggered flow `Auto_Create_Visit_Capture_On_Event` automatically creates a `Visit_Capture__c` whenever a user books an Event with an `AccountId`. Account/Contact/Opportunity are inherited. Result: **every meeting in the agenda becomes a visit to finalize** in the cockpit.

### ЁЯЦея╕П LWC experiences

| Component | Surface | Purpose |
|---|---|---|
| `vcVisitWorkspace` | Visit_Capture record page | Immersive 3-tab workspace (Preparation тЖТ Execution тЖТ Outcome) with AI actions |
| `vcCockpit` + 5 children | `/lightning/n/Visit_Cockpit` | Single-page dashboard: 6 KPIs, 2 action queues, analytics, recent visits, side drawer detail |
| `vcEventVisitCard` | Event record page (Related tab) | Inline Demo Context summary + AI Visit Brief for the visit linked to the Event тАФ no extra click |

All components are **fully bilingual** (English by default, French when the user's Language is set to French) via Custom Labels + translations.

## ЁЯУЛ Prerequisites

### Einstein Generative AI
- Einstein GenAI turned on
- Licensed edition (Enterprise, Unlimited, or Agentforce 1)
- Model available: **`sfdc_ai__DefaultGPT4Omni`** (used by the 3 prompts). On orgs without GPT-4 Omni, swap the model manually in Prompt Builder after deployment.

### Einstein Conversation Intelligence / In-Person Meeting (optional тАФ for ECI transcript path)

> Full ECI path is only required if you want automatic transcripts from mobile meetings. Without ECI, the component B still works with manual voice transcript (dictation).

**Licenses & editions**: Enterprise (EE), Unlimited (UE), Agentforce 1 (A1E). **Conversation Intelligence** must be licensed and enabled.

**Devices**:
| Platform | Minimum | Recommended |
|---|---|---|
| iOS | iOS 16+ | iPhone 15 Pro or newer |
| Android | 8 GB RAM, 6-core CPU, Android 13+ | Pixel 8+, Samsung S23+ |

**App**: sales reps must use **Sales Cloud Mobile** (not the generic Salesforce app).

**Admin setup (4 steps)**:
1. Setup тЖТ "Salesforce Go" тЖТ **Einstein Conversation Insights** тЖТ toggle ON
2. Setup тЖТ Permission Sets тЖТ assign **"Einstein Conversation Insights"** to mobile users
3. Setup тЖТ Salesforce Go тЖТ Einstein Conversation Insights тЖТ toggle **"In-Person Meetings"** ON
4. Setup тЖТ "Sales Cloud Mobile" тЖТ enable the app + distribute (App Store / MDM)

**Privacy**: transcription happens 100% on-device, audio is never transmitted, only the text surfaces in Salesforce. GDPR-compatible. **Not** HIPAA-compliant, **not** available on GovCloud.

**Languages** (driven by the device locale): FR, EN, ES, DE, IT, PT, JA, KO, AR, ZH, NL, PL, RU.

For an org **without ECI**, Component B still works in **standalone mode**: the rep pastes the voice transcript manually (or uses native mobile dictation) into `Voice_Transcript__c`. The prompt falls back automatically.

## ЁЯУж What gets deployed

### Custom objects
- **`Demo_Context__c`** тАФ 11 fields (Account_Name, Industry, Meeting_Type, Demo_Goal, Personas, Context_Notes + 5 AI fields)
- **`Visit_Capture__c`** тАФ 14 fields (Demo_Context, VideoCall, Event, Account, Contact, Opportunity + Voice_Transcript, Visit_Summary + 5 AI fields)

### Einstein Prompt Templates (`flex`, GPT-4 Omni)
- **`Prep_Demo_Grounding`** тАФ demo grounding generation (JSON output, 5 keys)
- **`Prepare_Visit_Brief`** тАФ pre-meeting HTML brief
- **`Generate_Visit_Report`** тАФ post-meeting report + action plan (JSON output)

All prompts use the `{!$User.LanguageLocaleKey}` merge field so the output follows the **invoking user's language**.

### Apex classes
- **`DemoGroundingParser`** тАФ invokes `Prep_Demo_Grounding`, parses JSON, writes 5 fields on Demo_Context
- **`VisitBriefGenerator`** тАФ invokes `Prepare_Visit_Brief`, writes HTML into `AI_Visit_Brief__c`
- **`VisitReportGenerator`** тАФ invokes `Generate_Visit_Report`, writes report + plan + language
- **`ActionPlanMaterializer`** тАФ parses the JSON action plan, creates Tasks + Events linked to the visit
- **`VisitCockpitController`** тАФ AuraEnabled controller backing all LWC (getSummary, getMissingReports, getRecentVisits, getDetail, saveField, regenerateReport, materializePlan, generateBrief, getVisitForEvent, createVisitForEvent, searchDemoContextsтАж)

### Flows
- **`Generate_Demo_Grounding`** (Screen) тЖТ invoked by Demo_Context Quick Action
- **`Generate_Visit_Report_Flow`** (Screen) тЖТ invoked by Visit_Capture Quick Action
- **`Materialize_Action_Plan`** (Screen) тЖТ invoked by Visit_Capture Quick Action
- **`VC_Grounding`** (PromptFlow, ECI only) тАФ data provider that fetches `VideoCallTranscript` linked to the visit's VideoCall and injects it into `Generate_Visit_Report`
- **`Auto_Create_Visit_Capture_On_Event`** (Record-Triggered, After Save on Create) тАФ auto-creates a Visit_Capture when an Event with `AccountId` is posted

### Quick Actions
- `Demo_Context__c.Generate_Grounding` тАФ "Generate Grounding"
- `Visit_Capture__c.Generate_Report` тАФ "Generate Report"
- `Visit_Capture__c.Materialize_Plan` тАФ "Materialize Action Plan"

### Lightning App + Tabs + FlexiPages
- **Custom App**: `Sales Visits` (FR: "Visites commerciales") with 4 tabs: Visit Cockpit, Visit Captures, Demo Contexts, Events
- **FlexiPages**:
  - `Visit_Cockpit_Page` тАФ hosts the single-page cockpit
  - `Visit_Capture_Record_Page` тАФ full-width record page with `vcVisitWorkspace`
  - `Event_Record_Page` тАФ patched with `vcEventVisitCard` in the Related tab (Demo Context summary + AI Visit Brief inline, no click required)

### Internationalization

- **Translations** (`fr.translation-meta.xml`): complete French translations for all UI strings
- **Object Translations**: `Demo_Context__c-fr`, `Visit_Capture__c-fr` (field labels, plural names)

### Permission Set

## тЪЩя╕П Post-deployment steps (mandatory)

### 1. Assign the permission set to your user
```bash
sf org assign permset --name Demo_Context_User --target-org <alias>
```

### 2. Activate the 3 prompt templates
Prompts land in **Draft** after deployment. Activate each:
1. Setup тЖТ **Prompt Builder**
2. Open `Prep_Demo_Grounding` тЖТ **Activate** (top-right)
3. Repeat for `Prepare_Visit_Brief` and `Generate_Visit_Report`

### 3. (ECI orgs) Verify the `VC_Grounding` flow
The flow deploys as **Active**, but if you see a newer Draft version in Flow Builder, make sure the right one is active.

### 4. Assign the `Sales Visits` app
Grant app visibility either via the permset (already done if step 1 passed) or Profile тЖТ Assigned Apps.

### 5. Activate the Visit Capture Workspace page
Setup тЖТ Lightning App Builder тЖТ open **Visit Capture Workspace** тЖТ **Activation** тЖТ "Org Default" (or assign to specific app/profile). Without this, opening a Visit Capture still shows the standard layout instead of the 3-tab workspace.

### 6. Activate the patched Event Record Page (**mandatory**)
The package ships an Event Record Page (`Event_Record_Page.flexipage-meta.xml`) that already embeds `vcEventVisitCard` in the Related tab тАФ so the **Demo Context summary + AI Visit Brief show up inline** the moment a sales rep opens an Event, with zero extra click.

But Salesforce never activates a deployed Lightning Record Page automatically. You must do it once:

1. Setup тЖТ **Lightning App Builder** тЖТ open **Event Record Page**
2. Click **ActivationтАж** тЖТ **Org Default** (or assign to the `Sales Visits` app + the relevant profiles)
3. **Save**
4. **Mobile:** if your org uses a separate **Phone** form-factor view, switch the App Builder canvas to it and confirm `vcEventVisitCard` is present (the bundle declares `Small` + `Large` form factors). Drop it manually if missing, then save and activate the Phone view.

Without this step, opening an Event shows the standard layout and the visit preparation card stays invisible.

### 7. (Optional) Switch a user to French to test i18n
Setup тЖТ Users тЖТ User тЖТ **Language = French** тЖТ logout/login тЖТ all UI strings and AI outputs switch to French seamlessly.

## ЁЯУЦ User guide

### Workflow A тАФ Prepare a demo
1. Open a **Demo Context** record (or create one)
2. Fill: account name, industry, meeting type, goal, personas, notes
3. Click **"Generate Grounding"** тЖТ the 5 AI fields populate in ~10 seconds
4. Review and adjust if needed

### Workflow B тАФ Capture a visit (3 tabs)

Open a Visit Capture. The `vcVisitWorkspace` LWC surfaces 3 tabs:

**Tab 1 тАФ Preparation**
- Pick the linked Demo Context (grounding pre-fills the AI context)
- Confirm Account / Contact / Opportunity
- Click **"Generate visit brief"** тЖТ a polished HTML brief appears (context, talking points, opening hooks, objections, success criteria)

**Tab 2 тАФ Execution**
- Link the VideoCall (if ECI is used) тАФ the transcript appears read-only
- Paste manual voice transcript and/or add free notes
- Upload attachments (jpg/png/pdf, max 5 MB each)
- Click **"Generate Report"** тЖТ report + plan are generated and you auto-land on tab 3
- Click **"Materialize Action Plan"** тЖТ real Tasks + Events appear in the CRM

**Tab 3 тАФ Outcome**
- Reads the HTML report, the action plan as cards, and the files
- Everything stays persistent тАФ reopening the record shows the latest state

### Workflow C тАФ Cockpit overview
Navigate to `/lightning/n/Visit_Cockpit` (or from the app). The cockpit shows:
- **6 KPIs** with deltas vs previous period (total visits, to-finalize, pending plans, tasks+events, ECI captured, manual captured)
- **2 action queues**: visits missing reports, plans pending materialization
- **Analytics**: weekly volume sparkline, source breakdown donut, top 5 accounts bar chart
- **Recent visits** table (last 10) with clickable rows opening the drawer
- **Side drawer** for quick report review / field autosave / AI regeneration without leaving the cockpit

## ЁЯза Design decisions

- **Rich Text Area** for `AI_Visit_Brief__c` and `AI_Report__c` тАФ the HTML renders natively in the standard UI, reports, and mobile app. No dedicated LWC needed for the detail page.
- **JSON sortie for plans** тАФ `AI_Action_Plan__c` keeps a structured payload so `ActionPlanMaterializer` can re-parse and create typed records (tasks vs events with priority, duration, due dates).
- **`{!$User.LanguageLocaleKey}` merge field** тАФ used in all 3 prompts to drive output language from the invoking user's locale, without needing `outputLanguageMode` (not yet fully available via Metadata API in all orgs).
- **Text(18) + Formula Hyperlink** for `Event__c` / `Event_Link__c` тАФ Salesforce disallows custom lookups pointing at `Event`, so the field holds the Id as text and a formula provides the clickable link.
- **LWC cockpit, not a report** тАФ the cockpit is a single-page LWC (no standard dashboard) because we need interactive drawers, autosave, AI re-generation, and custom visuals (donut, sparkline) in one immersive canvas.
